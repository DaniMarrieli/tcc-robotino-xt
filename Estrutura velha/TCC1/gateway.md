No seu cenário de revitalização do Robotino XT, o Raspberry Pi 4 assume o papel de **Gateway** e **Cliente** simultaneamente, sendo o ponto central da modernização. O **Robotino XT (Servidor)** é a máquina que hospeda o programa que usa a API nativa para controlar os motores e sensores, operando como um servidor de rede. A principal função do **Gateway (Raspberry Pi 4 / Cliente)** é atuar como um **tradutor de protocolos** e **ponte de comunicação bidirecional**. Ele **inicia a comunicação** (função de Cliente), conectando-se ao endereço IP do Robotino para solicitar e receber dados de odometria. O Gateway, que hospeda o _host_ ROS2, traduz a linguagem da API proprietária para as mensagens padronizadas do ROS2, e, por sua vez, recebe comandos de navegação do computador externo e os envia de volta ao Servidor para controle dos motores.
