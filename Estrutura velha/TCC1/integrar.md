A integração do Raspberry Pi 4 no Robotino XT é o elemento central da estratégia de modernização, estabelecendo uma **arquitetura de controle distribuída**. O Raspberry Pi 4 não substitui o sistema legado, mas atua como um **gateway** e a **unidade de processamento de alto nível**. Ele é configurado com um sistema operacional moderno (Ubuntu Server 22.04) rodando ROS2, resolvendo o problema de incompatibilidade de _drivers_ e _software_ do Robotino original. Sua principal função é criar uma **ponte de comunicação bidirecional**: recebendo dados de odometria do Robotino XT e publicando-os em tópicos ROS2, enquanto traduz comandos de navegação de volta para a API legada do robô.
